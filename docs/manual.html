详细文档

AssetLib233 全部配置项、运行时机制、构建流水线与扩展点的完整参考手册

1 PlayMode 详解

PlayMode 决定 AssetGroup 在编辑器或真机中如何定位和加载资源。在 AssetLib233PackageConfig 中为每个 AssetGroup 独立配置。

PlayMode 适用场景 说明
EditorSimulate 编辑器日常开发 不构建 AssetBundle，直接通过 AssetDatabase 模拟加载。速度最快，适合快速迭代；不支持真机运行。
EditorRemoteSimulation 编辑器联调远端 CDN 编辑器中加载远端服务器上的 Manifest 和 Bundle，用于在不打真机包的情况下验证 CDN 资源是否正确。
Host 真机开发 / 热更联调 运行时从配置的 DefaultHostServer 地址拉取 Manifest 和 Bundle。支持 FallbackHostServer 做容灾回退。
Web WebGL / 微信小游戏 通过 HTTP/HTTPS 从 CDN 或 Web 服务器加载资源，自动适配平台下载 API（如微信 WXAssetBundle 缓存链路）。
Offline 离线包 / 内置资源 仅从本地 StreamingAssets 或持久化缓存加载，不发网络请求。适用于首包内置资源或无网络环境。
提示：同一个 AssetGroup 在不同 PlayMode 下的 Manifest 解析逻辑完全一致，差异仅在 Bundle 文件的获取通道（本地 vs 网络 vs 编辑器模拟）。

2 AssetLib233PackageConfig 配置项

每个 AssetGroup 对应一份 AssetLib233PackageConfig，在编辑器 Inspector 或 JSON 中配置。

字段 类型 默认值 说明
PackageName string — AssetGroup 唯一标识，如 login、default、story。与构建产物的目录名和 Manifest 前缀一致。
PlayMode enum EditorSimulate 资源加载模式，详见上表。
DefaultHostServer string — 主 CDN / 服务器地址，如 https://cdn.example.com/res。运行时拼接 /{PackageName}/ 定位资源。
FallbackHostServer string — 备用服务器地址。当 DefaultHostServer 连接失败时自动回退。
EnableBundleCrypto bool false 启用后 Bundle 使用 XOR 加密写入，运行时通过 LoadFromStreamAsync 解密加载。
BundleCryptoPassword string "root" XOR 加密密钥。仅在 EnableBundleCrypto = true 时生效。
DownloadConcurrency int 10 该 AssetGroup 的最大并行下载数。推荐保持默认 10，过高的并发在弱网环境下可能引发超时。
GroupRole enum Optional Required：启动必需，下载完成后才可进入游戏。Optional：按玩法入口或后台低优先级下载。

3 Manifest 候选顺序

运行时初始化 AssetGroup 时，按以下顺序查找 Manifest 文件，命中即停：

  1. 原生 .manifest — AssetLib233 自有二进制格式，体积最小、解析最快。文件名：{PackageName}.manifest
  2. YooAsset .bytes — 兼容 YooAsset 1.x/2.x 的 Manifest 文件。文件名：{PackageName}.bytes
  3. PackageManifest 别名 — PackageManifest_{PackageName}.bytes 及 PackageManifest_{PackageName}.hash，用于无缝迁移 YooAsset CDN 产物。
迁移提示：如果你正在从 YooAsset 迁移，可以直接复用已有的 CDN 文件。AssetLib233 会自动识别并解析 YooAsset 格式的 Manifest，无需重新构建。

4 YooAsset CDN 兼容

AssetLib233 可以完全复用 YooAsset 的 CDN 产物文件，以下是所有被识别的文件类型：

文件后缀 / 名称 用途 兼容性
.version版本号文件，记录 Manifest 的版本哈希完全兼容
.manifestAssetLib233 原生 Manifest（二进制）原生
.bytesYooAsset Manifest 格式完全兼容
.hashPackageManifest 附属哈希文件完全兼容
.json构建报告 JSON仅编辑器
PackageManifest_*.bytesYooAsset 2.x PackageManifest 别名完全兼容
.bundleUnity AssetBundle 文件完全兼容
.rawRawBundle 原始文件（文本、音频等）完全兼容

5 XOR 加密说明

AssetLib233 内置基于 XOR 的轻量 Bundle 加密方案，适用于需要防止资源被直接解包的场景。

加密流程

  1. 构建管线在写入 Bundle 时，使用 BundleCryptoPassword（默认 "root"）对文件内容进行 XOR 运算
  2. 加密后的 Bundle 直接写入磁盘，无额外文件头
  3. 运行时通过 LoadFromStreamAsync 流式解密加载，支持 Seek 操作

配置示例

// AssetLib233PackageConfig 配置
EnableBundleCrypto = true,
BundleCryptoPassword = "my-secret-key"
注意：XOR 加密为轻量保护手段，不适合高强度安全需求场景。密钥请随项目配置管理，勿硬编码到开源仓库。

6 构建扩展点

AssetLib233 构建管线提供三个核心扩展接口，允许项目自定义打包行为：

接口 用途 典型实现
IAssetLib233BuildPackRule 定义资源如何分组和打包成 Bundle 按目录打包、按类型打包、按标签打包
IAssetLib233BuildCompressionStrategy 控制 Bundle 的压缩策略 LZ4、LZMA、Brotli、不压缩
IAssetLib233BuildVerifier 构建完成后的产物校验 Bundle 大小阈值检查、依赖完整性校验、Shader 变体覆盖校验

注册扩展

// 在 Editor 脚本中注册自定义扩展
AssetLib233EditorBuildPipeline.RegisterPackRule(new MyCustomPackRule());
AssetLib233EditorBuildPipeline.RegisterCompressionStrategy(new BrotliCompression());
AssetLib233EditorBuildPipeline.RegisterVerifier(new BundleSizeVerifier(50 * 1024 * 1024));

7 构建报告

每次构建完成后，AssetLib233 自动生成结构化报告，支持多种通知渠道。

JSON 报告

构建产物目录下生成 {PackageName}_build_report.json，包含：

  • 构建时间、Unity 版本、目标平台
  • Bundle 清单（文件名、大小、哈希、压缩率）
  • Collector 映射（Collector → Bundle 映射关系）
  • 变更差异（与上次构建的增删改对比）

飞书 Webhook 通知

设置环境变量 ASSETLIB233_FEISHU_WEBHOOK 为飞书机器人 Webhook URL，构建完成后自动推送报告摘要到飞书群。

# CI 环境中设置
export ASSETLIB233_FEISHU_WEBHOOK="https://open.feishu.cn/open-apis/bot/v2/hook/xxxx"

自定义通知

实现 IAssetLib233EditorBuildNotifier 接口可接入钉钉、Slack、邮件等任意通知渠道：

public class DingTalkNotifier : IAssetLib233EditorBuildNotifier
{
    public void OnBuildCompleted(BuildReport report)
    {
        // 发送钉钉群消息
    }
}

8 废弃 Bundle 清理

随着版本迭代，CDN 或本地缓存中会积累大量旧版本 Bundle。ObsoleteBundleCleaner 帮助安全清理这些废弃文件。

Dry-Run 模式

默认以 dry-run 模式运行，只输出将要删除的文件列表而不实际执行删除，方便人工确认：

// Editor 菜单或 CI 脚本中调用
ObsoleteBundleCleaner.Scan(
    packageConfig: config,
    dryRun: true  // 只输出不删除
);
// 确认后执行实际清理
ObsoleteBundleCleaner.Scan(
    packageConfig: config,
    dryRun: false
);

清理策略

  • 对比当前 Manifest 中的 Bundle 列表与 CDN / 本地目录中的实际文件
  • 不在当前 Manifest 中的文件被标记为 obsolete
  • 支持保留最近 N 个版本的 Bundle（通过配置 RetainVersionCount）

9 真机诊断

BuildRuntimeDiagnosticString 是排查真机资源加载问题的核心工具，一次性输出运行时的全部关键状态信息。

输出内容

  • 运行平台（RuntimePlatform）和 Unity 版本
  • 每个 AssetGroup 的 PlayMode、Manifest 版本哈希
  • CDN 地址（DefaultHostServer / FallbackHostServer）
  • 本地缓存路径和已缓存 Bundle 数量
  • 最近的下载事件（成功/失败/重试）和加载事件

使用方式

// 在问题发生时调用，将输出复制到 Issue 中
string diagnostic = AssetLib233.Instance.BuildRuntimeDiagnosticString();
Debug.Log(diagnostic);
建议：在真机调试 UI 中添加一个按钮，点击后调用此方法并复制到剪贴板，方便现场采集诊断信息。

10 Asset GC 配置

AssetLib233 内置资源垃圾回收机制，自动释放不再被引用的 Bundle 和 Asset。

配置项 类型 默认值 说明
GcMode enum Auto Auto：按间隔自动回收。Manual：仅手动调用 RunGc() 时回收。Disabled：关闭 GC。
GcIntervalSeconds float 60 Auto 模式下的自动 GC 间隔（秒）。
GracePeriodSeconds float 30 资源释放后的宽限期。引用归零后等待此时间再实际卸载，避免场景切换时反复加载。

手动触发

// 在场景切换完成后主动触发一次 GC
AssetLib233.Instance.RunGc();
注意：小游戏 / WebGL 运行时不建议直接调用 Resources.UnloadUnusedAssets()。资源释放应优先走 AssetLib233 的 GC 机制。

11 运行时插件

通过 IAssetLib233RuntimePlugin 接口，可以在不修改框架源码的前提下注入自定义的运行时行为观察器。

观察者事件

事件 触发时机
OnManifestLoadedAssetGroup 的 Manifest 加载完成
OnBundleDownloadStartedBundle 开始下载
OnBundleDownloadCompletedBundle 下载完成（含成功/失败状态）
OnAssetLoaded资源加载完成
OnAssetReleased资源被 GC 回收
OnError运行时出现错误（下载失败、解密失败、加载异常等）

注册插件

public class MyTelemetryPlugin : IAssetLib233RuntimePlugin
{
    public void OnBundleDownloadCompleted(string packageName, string bundleName, bool success)
    {
        // 上报到项目监控系统
    }
    // ... 其他事件
}

// 初始化时注册
AssetLib233.Instance.RegisterPlugin(new MyTelemetryPlugin());

12 AssetReference233 序列化引用

AssetReference233 是可序列化的资源引用类型，替代直接硬编码资源路径，支持 Inspector 拖拽赋值和运行时异步加载。

声明与使用

// MonoBehaviour 中声明
public AssetReference233 spriteRef;
public AssetReference233 prefabRef;

// 运行时加载
spriteRef.LoadAssetAsync<Sprite>((sprite) =>
{
    imageComponent.sprite = sprite;
});

优势

  • 解耦：不硬编码资源路径，通过 Inspector 拖拽绑定
  • 安全：内部持有 Collector 引用计数，自动跟踪生命周期
  • 跨 AssetGroup：支持引用不同 AssetGroup 中的资源
  • 编辑器预览：在 Inspector 中可直接预览引用的资源
