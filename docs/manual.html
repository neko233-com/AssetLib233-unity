构建产物与报告

核心只负责 AB、Manifest、version、加载、下载、GC 和诊断。构建完成后自动生成 JSON 报告并弹出报告窗口；报告默认保存在 Library/AssetLib233/BuildReports，不进 Git。

set ASSETLIB233_FEISHU_WEBHOOK=https://open.feishu.cn/open-apis/bot/v2/hook/xxx

飞书 webhook 只在环境变量存在时发送。项目也可注册 IAssetLib233EditorBuildNotifier 接入企业微信、钉钉、自建平台。

短地址与 XOR 加密

默认收集地址为文件名不带扩展名，Manifest 不再保存完整资源路径。运行时传入完整路径时会自动映射到短地址；构建端通过 AssetBundleBuild.addressableNames 保证 AB 内资源名与短地址一致。

AssetLib233 默认 XOR 密码为 root。项目可通过构建参数或运行时 AssetLib233PackageConfig.BundleCryptoPassword 覆盖；当前项目使用 neko233。构建端先记录明文 CRC，再对 AB 文件按偏移 XOR；运行端使用 LoadFromStreamAsync 流式解密，不整包复制。

构建扩展点

AssetLib233EditorBuildExtensionRegistry 支持注册 IAssetLib233BuildPackRule、IAssetLib233BuildCompressionStrategy、IAssetLib233BuildVerifier。压缩默认 LZ4，小游戏优先。

CDN 可选插件

Plugin_CDN_Deploy 已拆成可选插件，并通过 ASSETLIB233_CDN_DEPLOY 宏禁用。当前项目不启用它，继续使用已有 CDN 发布链路。

清理废弃 Bundle

AssetLib233ObsoleteBundleCleaner 根据最新 Manifest 和本地 CacheIndex 生成清理计划。可先 dry-run 输出报告，再执行删除，避免误删当前版本仍在引用的 AB。

真机诊断

string report = AssetLib233.Instance.BuildRuntimeDiagnosticString();
Debug.Log(report);

诊断串会输出平台、Package、Manifest 版本、缓存根路径、最近下载事件、Bundle 加载事件、错误信息。
