术语迁移表

从 YooAsset、Addressables 迁移到 AssetLib233 时，核心概念的对应关系。帮助团队快速理解新框架的命名与职责边界。

核心概念映射

下表列出 AssetLib233 中的关键类型 / 概念，以及在 YooAsset 和 Addressables 中的对应物。"迁移说明"列描述差异点和注意事项。

AssetLib233 YooAsset Addressables 迁移说明
AssetGroup Package Catalog / Group 概念一致：一组可独立更新、独立下载的资源集合。AssetLib233 原生支持 login / default / story / voice 等多组分层的启动链路。
AssetCollector Collector Asset Group Template 负责将编辑器资源收集到 Bundle。AssetLib233 的 Collector 支持更多打包规则（按目录、按标签、按类型、自定义），且与构建报告深度集成。
AssetManifest233 PackageManifest content_catalogs 二进制 manifest，包含每个 Bundle 的 hash、size、deps、地址映射。AssetLib233 使用紧凑二进制格式，解析速度更快，内存占用更低。
PackageVersion PackageVersion remote catalog hash 轻量文本文件，记录远端最新 manifest 版本号。运行时先拉取此文件判断是否需要更新。
ContentPack
业务层		 无直接对应 无直接对应 业务层概念，指热更层启动后管理的各 AssetGroup。按优先级分为 Required（必须完成）和 Optional（后台下载）。这是 AssetLib233 独有的业务分层抽象。
UnifiedLoadingProgress 需业务实现 PercentComplete（单 Group） 多 AssetGroup 下载时，将各组的 bytes / files 汇总为一个 0~1 进度值。Addressables 只有单 Group 进度，YooAsset 需要业务自行聚合。
AssetHandle233<T> AssetOperationHandle AsyncOperationHandle<T> 异步加载句柄，支持 await、引用计数和显式 Release()。泛型版本 AssetHandle233<T> 在编译期即可确定资源类型。
AssetLib233PackageConfig PackageParameters Profile / Build Script 运行时配置：CDN 地址、PlayMode、加密选项等。比 YooAsset 的 PackageParameters 增加了 FallbackHostServer 和 BundleCrypto 字段。
PreparePackageOperation InitializationOperation InitializeAsync 拉取远端 version → 下载远端 manifest → 与本地 manifest 比对差异。返回是否需要下载以及差异 Bundle 列表。
PackageDownloadOperation UpdatePackageManifestOperation + PreDownloadContentOperation DownloadDependenciesAsync 基于 Prepare 阶段的差异列表执行增量下载。内置并发调度（默认 10）、断点续传、失败重试和进度回调。YooAsset 需要两步操作。
AssetGcService Package.UnloadUnusedAssets Resources.UnloadUnusedAssets 独立 GC 服务，按引用计数回收 Bundle。支持自动周期扫描和手动触发，可针对不同 Group 独立 GC。比 Addressables 的全局释放粒度更细。
Platform Plugin
NEW		 无 无 平台插件层，为微信小游戏、WebGL 等提供专属的下载 / 缓存 / 诊断能力。如微信小游戏的 WXAssetBundle 缓存链路、CDN 预热等，无需改核心代码。

CDN 文件命名映射

不同框架在 CDN 上使用的文件命名规则不同。下表帮助运维 / CI 同学理解迁移后的文件结构变化。

文件用途 AssetLib233 YooAsset Addressables
远端版本 {GroupName}.version {PackageVersion}.version catalog hash 内嵌在 settings
清单文件 {GroupName}_{Hash}.manifest {PackageName}_{Hash}.bytes catalog_{Hash}.json
Bundle 文件 {BundleHash}.bundle {BundleHash}.bundle {Hash}_{Name}.bundle
加密 Bundle {BundleHash}.bundle.enc 无标准后缀 无
构建报告 {GroupName}_BuildReport.json 无标准格式 AddressablesBuildReport.json

AssetLib233 的 Bundle 文件名使用 hash 命名，天然支持内容寻址缓存（content-addressable cache），相同内容不会重复下载。加密 Bundle 使用 .enc 后缀以便 CDN 和运维识别。

PlayMode 映射

三个框架都提供多种运行模式，用于不同开发和测试阶段。下表列出对应关系。

场景 AssetLib233 YooAsset Addressables
编辑器内快速迭代 EditorSimulate EditorSimulateMode Use Asset Database
本地打包测试 OfflinePlay OfflinePlayMode Use Existing Build
CDN 热更 Host HostPlayMode Simulate Groups / 手动
微信小游戏模拟 WxSimulate
NEW		 无 无

模式选择建议

  • 开发阶段 — 使用 EditorSimulate，不打包即可加载编辑器资源，迭代最快
  • 集成测试 — 使用 OfflinePlay，验证打包产物和 Bundle 依赖是否正确
  • 热更验证 — 使用 Host，从 CDN 拉取最新资源，验证增量下载和版本比对
  • 小游戏调试 — 使用 WxSimulate，模拟微信小游戏缓存链路和 WXAssetBundle 行为

迁移注意事项

  • 地址格式 — AssetLib233 的资源地址默认使用完整的 Asset 路径（如 Assets/Prefabs/hero.prefab），与 YooAsset 的短地址模式不同。如需短地址，在 Collector 中启用 AddressByFileName 规则。
  • 初始化时序 — AssetLib233 需要先 Initialize() 再 InitializeGroup(config)，而 YooAsset 的 YooAssets.Initialize() 内已包含 Package 注册。迁移时需拆为两步。
  • 引用释放 — AssetHandle233.Release() 是显式递减引用计数，计数归零后由 AssetGcService 实际卸载。不要在 Release() 后立即假设 Bundle 已卸载。
  • SubAssets — LoadSubAssetsAsync<T> 对应 YooAsset 的 LoadSubAssetsAsync，返回数组。Addressables 需使用 LoadAssetsAsync + Label 过滤。
  • Raw File — LoadRawFileAsync 加载非 AB 格式的原始文件（如 JSON、XML、二进制配置），返回 byte[] 或 string。
