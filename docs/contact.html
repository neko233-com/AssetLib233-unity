2 Issue 反馈模板

提交 Issue 时请尽量包含以下信息，帮助我们快速定位问题：

Issue 模板 — Bug 报告

运行平台 如：微信小游戏 Android / iOS / PC Windows
AssetGroup 出问题的 AssetGroup 名称，如 login / default / story
PlayMode 如 Host / Web / Offline / EditorSimulate
Manifest 版本 诊断输出中的 Manifest 哈希值
CDN 地址 DefaultHostServer / FallbackHostServer 配置
CDN URL 实际请求的 Bundle 完整 URL
诊断字符串 BuildRuntimeDiagnosticString() 的完整输出
复现步骤 详细的复现步骤描述

示例

## 平台
微信小游戏，Android 真机（微信 8.0.x）

## AssetGroup
default

## PlayMode
Web

## Manifest 版本
a3f8c2d1e5b7

## CDN 地址
https://cdn.example.com/res/default/

## 诊断字符串
（粘贴 BuildRuntimeDiagnosticString() 输出）

## 复现步骤
1. 启动游戏
2. 登录成功后进入主城
3. default 包下载进度卡在 45%，Console 无报错

## 期望行为
default 包应完整下载并进入主城

## 实际行为
下载卡住，Loading 动画持续播放
提示：诊断字符串是定位问题的关键。请在问题发生时立即调用 BuildRuntimeDiagnosticString() 并粘贴完整输出。

3 贡献指南

我们欢迎各种形式的贡献，无论是 Bug 修复、功能增强还是文档改进。

贡献流程

  1. Fork — Fork 仓库到个人 GitHub 账号
  2. Branch — 从 main 创建功能分支（如 feature/xxx 或 fix/xxx）
  3. Develop — 编写代码并确保本地测试通过
  4. PR — 提交 Pull Request，描述修改内容和原因
  5. Review — 等待代码审查和合并

编码规范

规则 说明
语言 C# 代码，注释使用简体中文
命名 PascalCase（类/方法/属性），camelCase（局部变量），_camelCase（私有字段）
类型声明 显式类型，避免隐式 var
循环 热路径使用 for 循环，避免 foreach（减少 GC）
线程 单线程主循环模型，禁止 Thread / Task.Run / 并发容器
异步 使用 UniTask 或原生 AsyncOperation，禁止 Task.Run
资源加载 统一走 AssetLib233.Instance API，禁止绕过框架直接调用底层
平台宏 平台相关代码必须用 #if 宏完整包裹，不留裸引用

PR 检查清单

  • 代码能通过编译，无 warning
  • 新增 API 有对应的文档更新
  • 涉及运行时的修改有对应的测试覆盖或验证说明
  • 不涉及 .meta 文件的手动修改
  • PR 描述清晰说明修改目的和影响范围
