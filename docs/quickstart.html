快速上手

从安装到构建，7 步跑通完整热更链路。每步只需复制粘贴示例代码，即可在项目中使用。

1 安装

AssetLib233 提供两种安装方式，任选其一。

方式 A — UPM Git URL（推荐）

在 Unity 编辑器中打开 Window → Package Manager → Add package from git URL，粘贴：

https://github.com/neko233-com/AssetLib233-unity.git

方式 B — PowerShell 一键导入

无 Git 环境时，可在 Unity 的 Package Manager Console 或系统终端执行：

powershell -ExecutionPolicy Bypass -Command "iwr https://raw.githubusercontent.com/neko233-com/AssetLib233-unity/main/Tools/install-assetlib233.ps1 -OutFile $env:TEMP/install-assetlib233.ps1; & $env:TEMP/install-assetlib233.ps1 -ProjectRoot (Get-Location).Path"

安装完成后，在 Project Settings 中可看到 AssetLib233 配置面板。

2 初始化

在游戏启动脚本中完成初始化。先调用全局 Initialize()，再为每个 AssetGroup 创建配置并注册。

// 全局初始化（只需调用一次）
AssetLib233.Instance.Initialize();

// 为 "default" 组创建配置
AssetLib233PackageConfig config = new AssetLib233PackageConfig();
config.PackageName          = "default";
config.PlayMode             = EnumHotUpdateType.Host;
config.DefaultHostServer    = "https://cdn.example.com/default";
config.FallbackHostServer   = "https://backup.example.com/default";
config.EnableBundleCrypto   = true;

// 注册到 AssetLib233
AssetLib233.Instance.InitializeGroup(config);

配置字段说明

  • PackageName — AssetGroup 唯一标识，对应构建时的分组名
  • PlayMode — Host（CDN 热更）、EditorSimulate（编辑器模拟）、OfflinePlay（本地沙盒）
  • DefaultHostServer — 主 CDN 地址，运行时从此拉取 version 与 manifest
  • FallbackHostServer — 备用 CDN，主地址失败时自动回退
  • EnableBundleCrypto — 启用 Bundle 加密，防止资源被直接提取

3 热更准备与下载

初始化完成后，先执行 PreparePackageOperation（拉取远端版本号 + 比对本地 Manifest），再执行 PackageDownloadOperation（增量下载差异 Bundle）。

// 第一步：准备 — 拉取远端 version，比对本地 manifest
AssetLib233PreparePackageOperation prepareOp =
    AssetLib233.Instance.CreatePreparePackageOperation("default");
await prepareOp;

// 检查准备是否成功
if (prepareOp.Status != EnumOperationStatus.Succeed)
{
    Debug.LogError($"Prepare failed: {prepareOp.Error}");
    return;
}

// 第二步：下载 — 增量下载差异 Bundle
AssetLib233PackageDownloadOperation downloadOp =
    AssetLib233.Instance.CreatePackageDownloadOperation("default", string.Empty);
await downloadOp;

// 监听下载进度（可选）
downloadOp.OnProgressChanged += (downloaded, total, ratio) =>
{
    Debug.Log($"下载进度: {ratio:P1} ({downloaded}/{total})");
};

PreparePackageOperation 会自动处理首次安装（无本地缓存）和增量更新两种场景。下载操作内部已做并发调度（默认并发上限 10）与失败重试。

4 加载资源

资源下载完成后，通过 LoadAssetAsync<T> 异步加载。返回的 AssetHandle233<T> 支持 await，加载完成后通过 handle.AssetObject 取出资源。

// 异步加载 Prefab
AssetHandle233<GameObject> handle =
    AssetLib233.Instance.LoadAssetAsync<GameObject>("default", "Assets/Prefabs/main.prefab");
await handle;

// 取出加载结果
GameObject prefab = handle.AssetObject;
if (prefab != null)
{
    Object.Instantiate(prefab);
}

// 加载 Sprite 示例
AssetHandle233<Sprite> spriteHandle =
    AssetLib233.Instance.LoadAssetAsync<Sprite>("default", "Assets/Sprites/hero_icon.png");
await spriteHandle;
myImage.sprite = spriteHandle.AssetObject;

加载地址支持 AssetBundle 原始路径或 Collector 配置的短地址，具体取决于构建时的收集器设置。

5 多组下载（统一 Loading）

典型项目中，登录前只需下载 login 首组，登录成功后再下载 default、story 等内容组。MultiGroupDownloadOperation 可将多个 AssetGroup 的下载进度汇总为一个统一的 Loading 进度。

// 定义启动计划
AssetLib233StartupPlan plan = new AssetLib233StartupPlan();
plan.LoginGroupName = "login";

// 登录后必须完成的组（Required）
plan.PostLoginRequiredGroups.Add("default");
plan.PostLoginRequiredGroups.Add("story");

// 登录后可选后台下载的组（Optional）
plan.PostLoginOptionalGroups.Add("voice");
plan.PostLoginOptionalGroups.Add("cg");

// 创建多组统一下载操作
AssetLib233MultiGroupDownloadOperation multiOp =
    AssetLib233.Instance.CreateMultiGroupDownloadOperation(plan);

// 统一进度回调
multiOp.OnProgressChanged += (downloaded, total, ratio) =>
{
    loadingBar.SetProgress(ratio);
};

await multiOp;

StartupPlan 分组策略

  • LoginGroupName — 首包，登录 UI 可见前必须完成，体尽量小
  • RequiredGroups — 进入主城 / 核心玩法前必须完成的组，Loading 会硬等
  • OptionalGroups — 后台低优先级下载，不阻塞玩家操作

6 释放资源

使用完毕的资源需要通过 Handle 手动释放，或交给 AssetGcService 自动回收。AssetLib233 内部维护引用计数，确保同一个 Bundle 在所有引用释放后才会被卸载。

手动释放

// 加载并使用完毕后释放
AssetHandle233<GameObject> handle =
    AssetLib233.Instance.LoadAssetAsync<GameObject>("default", "Assets/Prefabs/monster.prefab");
await handle;

GameObject go = Object.Instantiate(handle.AssetObject);
// ... 使用完毕后 ...
handle.Release();

自动 GC

// 开启自动 GC 服务（建议在初始化时调用一次）
AssetLib233.Instance.StartAssetGcService(intervalSeconds: 60f);

// 手动触发一次 GC（如场景切换时）
AssetLib233.Instance.AssetGcCollect("default");

AssetGcService 会周期性扫描所有 AssetGroup 的引用计数，将计数为 0 的 Bundle 卸载并释放内存。小游戏 / WebGL 平台建议配合 Resources.UnloadUnusedAssets() 使用，但需注意调用频率避免卡顿。

7 构建

AssetLib233 提供编辑器菜单和 API 两种构建入口。

方式 A — 编辑器菜单

在 Unity 菜单栏找到 AssetLib233 / 构建 AssetBundle，选择 BuildProfile 后一键构建。构建完成后会自动弹出构建报告窗口，包含：

  • 各 AssetGroup 的 Bundle 数量、总大小、加密状态
  • 新增 / 变更 / 删除 Bundle 清单
  • CDN 文件命名映射
  • 错误与警告摘要

方式 B — API 调用

// 在 Editor 脚本或 CI 中调用
AssetLib233EditorBuildPipeline.BuildAllGroups(buildProfile);

// 构建产物输出到 StreamingAssets 和 CDN 目录
// 可选：设置飞书 Webhook 环境变量启用构建通知
// ASSETLIB233_FEISHU_WEBHOOK=https://open.feishu.cn/open-apis/bot/v2/hook/xxx

CDN 发布

构建完成后，将 CDN 输出目录中的文件上传到 DefaultHostServer 对应的 CDN 路径。上传时保持目录结构不变，.version 文件必须在根目录。CDN 需支持 Range 请求以启用断点续传。
