常见问题与解答 — 涵盖平台适配、配置调优、迁移排障等高频场景

Q
微信小游戏需要哪些编译宏？
平台适配

微信小游戏环境下，AssetLib233 依赖以下编译宏：

  • WX — 核心微信小游戏宏，由团结引擎微信小游戏构建自动定义
  • WEIXINMINIGAME — 微信小游戏平台标识宏

这两个宏控制微信特有的下载通道（WXAssetBundle 预下载缓存链路）、平台缓存判断和网络预热逻辑。所有 #if WX 块内的类型引用、方法签名和字段声明都必须被完整包裹，避免 Windows 打包时出现 CS0246 错误。

Q
Editor 下正常，真机资源为空怎么办？
运行时排障

这是最常见的真机问题。按以下步骤排查：

  1. 调用 AssetLib233.Instance.BuildRuntimeDiagnosticString()，获取完整诊断信息
  2. 检查诊断输出中的 PlayMode 是否正确 — 真机应为 Host、Web 或 Offline，而非 EditorSimulate
  3. 检查 Manifest 版本哈希 是否与 CDN 上一致
  4. 检查 CDN 地址 是否正确拼接了 /{PackageName}/ 路径
  5. 确认 Bundle 文件确实存在于 CDN 上（可用浏览器直接访问 URL 验证）
常见原因：CI 打包后 CDN 未刷新缓存，导致真机拉到旧 Manifest。
Q
多个 AssetGroup 怎么显示一个统一的 Loading？
运行时

使用 CreateMultiGroupDownloadOperation 将多个 AssetGroup 的下载进度合并为统一操作：

var operation = AssetLib233.Instance.CreateMultiGroupDownloadOperation(
    "default", "story"
);
operation.OnProgressUpdated += (progress) =>
{
    // progress 为所有组的加权平均进度 (0~1)
    loadingBar.SetProgress(progress);
};
await operation.StartAsync();

该方法自动按各 AssetGroup 的 Bundle 总大小做加权进度计算，避免某一组体积小导致进度条跳变。

Q
可以不用 UniTask 吗？
配置

可以。AssetLib233 的运行时 API 原生支持两种异步模式：

  • Callback 模式：传入回调函数，零依赖
  • await 原生：返回 AsyncOperation 兼容对象，可用 await 等待

UniTask 作为可选增强，提供 AsUniTask() 扩展方法。不引入 UniTask 包时，该扩展方法不可用但不影响核心功能。

// Callback 模式
AssetLib233.Instance.LoadAssetAsync<Sprite>("path", OnSpriteLoaded);

// await 模式（原生）
var op = AssetLib233.Instance.LoadAssetAsync<Sprite>("path");
await op;
var sprite = op.Result;
Q
CDN 发布在哪里做？
构建

AssetLib233 不内置 CDN 上传功能。构建管线只负责生成产物（Bundle、Manifest、Version 文件等），CDN 发布由项目自有的 CI/CD 流水线处理。

推荐做法：

  • 构建完成后，产物输出到指定目录（如 BuildOutput/{Platform}/{PackageName}/）
  • CI 脚本使用云厂商 CLI 工具（如 aliyun oss cp、aws s3 sync）将产物同步到 CDN
  • 通过 IAssetLib233EditorBuildNotifier 在构建完成后自动触发 CI 上传任务
Q
如何从 YooAsset 迁移到 AssetLib233？
迁移

迁移分两步：

  1. 运行时 API 替换 — 将 YooAssets / ResourcePackage 调用替换为 AssetLib233.Instance 对应方法。术语映射参考术语迁移页面。
  2. CDN 产物兼容 — AssetLib233 原生支持 YooAsset 的 CDN 文件格式（.bytes、.hash、PackageManifest_*），可以直接复用已有 CDN 文件，无需重新构建。
渐进迁移：可以逐个 AssetGroup 迁移，先迁 login 包验证链路，再迁 default/story 等内容包。

Q
下载并发可以配置吗？
配置

可以。每个 AssetGroup 的 AssetLib233PackageConfig.DownloadConcurrency 控制最大并行下载数。

  • 默认值：10
  • 推荐值：10（过高并发在弱网下可能引发超时）
  • 微信小游戏下 WXDownloadSettings.SetMaxConcurrent(...) 也应传 10

如果真机弱网出现抖动，优先做重试、断点续传和进度体验优化，不建议通过增大并发来硬冲。

Q
如何处理加密 Bundle？
运行时

AssetLib233 内置 XOR 加密方案：

  1. 在 AssetLib233PackageConfig 中设置 EnableBundleCrypto = true
  2. 配置 BundleCryptoPassword（默认 "root"）
  3. 构建管线自动对产出的 Bundle 做 XOR 加密
  4. 运行时通过 LoadFromStreamAsync 流式解密加载，支持 Seek 操作

加密密钥通过配置文件管理，请勿硬编码到开源仓库中。

Q
Asset GC 什么时候触发？
运行时

取决于 GcMode 配置：

  • Auto（默认）：按 GcIntervalSeconds（默认 60 秒）自动触发一轮 GC
  • Manual：仅手动调用 AssetLib233.Instance.RunGc() 时回收
  • Disabled：关闭 GC

推荐在场景切换完成后主动调用一次 RunGc()，配合 Auto 模式做兜底。GracePeriodSeconds（默认 30 秒）是资源释放后的宽限期，避免场景切换时反复加载。

Q
如何清理旧版本缓存？
构建

使用 ObsoleteBundleCleaner：

// 先 dry-run 查看将要删除的文件
ObsoleteBundleCleaner.Scan(config, dryRun: true);
// 确认后执行实际清理
ObsoleteBundleCleaner.Scan(config, dryRun: false);

支持通过 RetainVersionCount 保留最近 N 个版本的 Bundle，避免一次性清理所有历史文件。

编辑器模式和真机模式有什么区别？
配置
模式编辑器真机
EditorSimulate直接模拟加载，无需构建 AB不可用
EditorRemoteSimulation编辑器加载远端 CDN 资源不可用

核心区别在于 Bundle 文件的获取通道不同，Manifest 解析和资源加载逻辑完全一致。

Q
如何自定义下载传输层？
扩展

实现 IAssetLib233DownloadTransport 接口：

public class CustomTransport : IAssetLib233DownloadTransport
{
    public void Download(string url, string savePath,
        Action<float> onProgress, Action<bool, string> onComplete)
    {
        // 自定义下载实现（如 P2P、自定义协议等）
    }
}

// 注册
AssetLib233.Instance.SetDownloadTransport(new CustomTransport());

适用于需要对接特殊下载通道（如游戏加速器、P2P 传输等）的场景。

Q
如何添加新平台支持？
扩展

实现 IAssetLib233PlatformPlugin 接口：

public class MyPlatformPlugin : IAssetLib233PlatformPlugin
{
    public string PlatformName => "MyPlatform";

    public IAssetLib233DownloadTransport CreateDownloadTransport()
    {
        return new MyPlatformTransport();
    }

    public string GetCacheRootPath()
    {
        return "/path/to/platform/cache";
    }

    public bool IsBundleCached(string bundleName, string hash)
    {
        // 平台特有的缓存判断逻辑
    }
}

// 注册
AssetLib233.Instance.RegisterPlatformPlugin(new MyPlatformPlugin());

该接口抽象了平台差异，包括下载通道、缓存路径和缓存判断。

Q
构建产物有哪些文件？
构建
文件说明用途
{PackageName}.version文本文件，记录 Manifest 版本哈希运行时判断是否需要更新
{PackageName}.manifest二进制 Manifest（AssetLib233 格式）Bundle 索引、依赖关系、哈希校验
*.bundleUnity AssetBundle 文件实际的资源包
*.rawRawBundle 原始文件文本、音频等非 AB 资源
{PackageName}_build_report.json构建报告产物清单、变更差异、大小统计

所有文件输出到 BuildOutput/{Platform}/{PackageName}/ 目录下。

Q
飞书通知怎么配置？
构建

设置环境变量 ASSETLIB233_FEISHU_WEBHOOK 即可：

# Linux / macOS
export ASSETLIB233_FEISHU_WEBHOOK="https://open.feishu.cn/open-apis/bot/v2/hook/xxxx"

# Windows PowerShell
$env:ASSETLIB233_FEISHU_WEBHOOK = "https://open.feishu.cn/open-apis/bot/v2/hook/xxxx"

构建完成后，AssetLib233 自动向飞书群推送报告摘要（包含平台、Bundle 数量、总大小、变更差异）。如果环境变量未设置则跳过通知，不影响构建流程。

如需接入钉钉、Slack 等其他通知渠道，实现 IAssetLib233EditorBuildNotifier 接口即可。
